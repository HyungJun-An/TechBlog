가중치가 다른 이동 방식(순간이동 0초, 걷기 1초)이 포함된 최단 시간 찾기 문제입니다. 0-1 BFS(Deque 활용)로 해결하는 과정에서 겪은 오류와 핵심 인사이트를 공유합니다.

<!-- truncate -->

## 문제 설명

수빈이는 위치 X에서 동생을 찾아야 합니다.

- **걷기**: 1초 후 X-1 또는 X+1로 이동
- **순간이동**: 0초 후 2×X로 이동

예: 2 → 5 로 가는 최단 경로
`2 → 4 (순간이동, 0초) → 5 (걷기, 1초)` = **1초**

[백준 13549번](https://www.acmicpc.net/problem/13549)

## 초기 접근 (오답)

처음에는 일반 `Queue`를 사용해 모든 이동을 동일하게 처리했습니다.

```java
Queue<int[]> q = new LinkedList<>();
q.add(new int[]{N, 0});

while (!q.isEmpty()) {
    int[] cur = q.poll();
    int pos = cur[0], cnt = cur[1];

    if (pos == K) {
        System.out.println(cnt);
        return;
    }

    int twice = pos * 2;
    int plus  = pos + 1;
    int minus = pos - 1;

    if (twice <= MAX && !visited[twice]) {
        visited[twice] = true;
        q.add(new int[]{twice, cnt});      // 0초
    }
    if (plus <= MAX && !visited[plus]) {
        visited[plus] = true;
        q.add(new int[]{plus, cnt + 1});   // 1초
    }
    if (minus >= 0 && !visited[minus]) {
        visited[minus] = true;
        q.add(new int[]{minus, cnt + 1});  // 1초
    }
}
```

**왜 틀렸나?**

`1 → 2`로 이동할 때 두 경로가 있습니다.
- `1 + 1 = 2` (걷기, 1초)
- `1 × 2 = 2` (순간이동, 0초)

일반 Queue는 FIFO이므로 먼저 삽입된 경로가 먼저 처리됩니다. 순간이동(0초)보다 걷기(1초)가 먼저 큐에 들어가면, 걷기로 2를 방문했다고 체크하고 나중에 들어오는 순간이동(0초) 경로를 무시합니다.

## 핵심 인사이트: 0-1 BFS

이동 비용이 **0 또는 1** 두 종류일 때, `Deque`를 활용해 우선순위를 구현할 수 있습니다.

- **비용 0 이동 (순간이동)**: 덱의 **앞**에 추가 (`addFirst`)
- **비용 1 이동 (걷기)**: 덱의 **뒤**에 추가 (`addLast`)

이렇게 하면 다익스트라처럼 비용이 낮은 경로가 항상 먼저 처리됩니다.

## 최종 풀이 코드

```java
import java.util.*;
import java.io.*;

public class Main {
    static final int MAX = 100001;

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int N = Integer.parseInt(st.nextToken()); // 수빈이 위치
        int K = Integer.parseInt(st.nextToken()); // 동생 위치

        int[] dist = new int[MAX];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[N] = 0;

        Deque<Integer> dq = new ArrayDeque<>();
        dq.addFirst(N);

        while (!dq.isEmpty()) {
            int cur = dq.pollFirst();

            if (cur == K) {
                System.out.println(dist[K]);
                return;
            }

            // 순간이동: 비용 0 → 덱 앞에 추가
            int twice = cur * 2;
            if (twice < MAX && dist[twice] > dist[cur]) {
                dist[twice] = dist[cur];
                dq.addFirst(twice);
            }

            // 걷기 (+1, -1): 비용 1 → 덱 뒤에 추가
            int[] next = {cur + 1, cur - 1};
            for (int nx : next) {
                if (nx >= 0 && nx < MAX && dist[nx] > dist[cur] + 1) {
                    dist[nx] = dist[cur] + 1;
                    dq.addLast(nx);
                }
            }
        }

        System.out.println(dist[K]);
    }
}
```

## 핵심 포인트

### 방문 체크 시점
초기 오답 코드에서는 **큐에 삽입할 때** 방문 체크를 했습니다. 이 경우 더 빠른 경로로 같은 노드에 도달하더라도 이미 방문 처리되어 무시됩니다.

정답 코드에서는 **거리 비교(`dist[nx] > dist[cur] + cost`)** 방식을 사용해, 더 짧은 경로가 발견되면 업데이트하고 덱에 다시 추가합니다.

### 0-1 BFS vs 다익스트라

| 방법 | 시간복잡도 | 사용 조건 |
|------|-----------|-----------|
| 0-1 BFS | O(V + E) | 가중치가 0 또는 1 |
| 다익스트라 | O((V + E) log V) | 가중치가 음수가 아닌 임의 값 |

이 문제는 가중치가 0과 1뿐이므로 0-1 BFS가 더 효율적입니다.

## 배운 점

일반 BFS는 모든 간선의 가중치가 동일할 때만 올바르게 동작합니다. 가중치가 다른 경우에는 반드시 우선순위를 고려해야 합니다. 가중치가 0과 1인 특수한 케이스에서는 `Deque`로 간단하게 다익스트라와 동일한 효과를 낼 수 있습니다.
