BFS 탐색 시 "벽을 부순 횟수"를 상태값에 포함하는 3차원 visited 배열을 활용한 최단경로 문제입니다. 초기 오답 분석과 함께 핵심 인사이트를 정리합니다.

<!-- truncate -->

## 문제 설명

N×M 격자에서 (1,1)에서 (N,M)으로 이동하는 최단 경로를 구합니다.

- 이동: 상하좌우 인접 칸, 이동마다 거리 +1
- 특수 조건: 벽(1)을 **최대 K번**까지 부수고 지나갈 수 있음

[백준 14442번](https://www.acmicpc.net/problem/14442)

## 초기 오답 분석

처음에 Deque로 벽 부수기를 0-1 BFS처럼 처리하려 했습니다.

```java
// 잘못된 접근: 벽 부수기를 비용 0으로 처리
if (map[nx][ny] == 1 && breaksLeft > 0) {
    dq.addFirst(new int[]{nx, ny, dist + 1, breaksLeft - 1});
} else {
    dq.addLast(new int[]{nx, ny, dist + 1, breaksLeft});
}
```

**왜 틀렸나?**

모든 이동의 비용이 동일하게 1입니다. 벽을 부수는 것도 비용 1, 그냥 이동하는 것도 비용 1입니다. 0-1 BFS는 비용 차이가 있을 때만 의미 있습니다.

진짜 문제는 이것입니다: **같은 위치에 다른 "남은 벽 부수기 횟수"로 도달하는 경우**를 구분해야 합니다.

```
예시:
A → B (벽 안 부수고 도달, breaksLeft=2)
C → B (벽 1개 부수고 도달, breaksLeft=1)
```

`visited[B] = true`로만 체크하면, 먼저 도달한 경로(예: breaksLeft=1)가 B를 방문 처리해서 breaksLeft=2로 오는 더 유리한 경로를 차단해버립니다.

## 핵심: 3차원 visited 배열

해결 방법은 **(위치 + 남은 벽 부수기 횟수)를 하나의 상태**로 봐야 한다는 것입니다.

```java
boolean[][][] visited = new boolean[N][M][K + 1];
// visited[row][col][breaks_remaining]
```

같은 칸이라도 남은 횟수가 다르면 다른 상태입니다.

## 최종 풀이 코드

```java
import java.util.*;
import java.io.*;

public class Main {
    static int[] dx = {0, 1, 0, -1};
    static int[] dy = {1, 0, -1, 0};

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int N = Integer.parseInt(st.nextToken());
        int M = Integer.parseInt(st.nextToken());
        int K = Integer.parseInt(st.nextToken());

        int[][] map = new int[N][M];
        for (int i = 0; i < N; i++) {
            String line = br.readLine();
            for (int j = 0; j < M; j++) {
                map[i][j] = line.charAt(j) - '0';
            }
        }

        // 상태: (행, 열, 남은 벽 부수기 횟수)
        boolean[][][] visited = new boolean[N][M][K + 1];
        Queue<int[]> q = new LinkedList<>();

        // {행, 열, 거리, 남은 벽 부수기 횟수}
        q.add(new int[]{0, 0, 1, K});
        visited[0][0][K] = true;

        while (!q.isEmpty()) {
            int[] cur = q.poll();
            int x = cur[0], y = cur[1], dist = cur[2], breaksLeft = cur[3];

            // 목적지 도달
            if (x == N - 1 && y == M - 1) {
                System.out.println(dist);
                return;
            }

            for (int d = 0; d < 4; d++) {
                int nx = x + dx[d];
                int ny = y + dy[d];

                if (nx < 0 || nx >= N || ny < 0 || ny >= M) continue;

                if (map[nx][ny] == 0) {
                    // 빈 칸: 그냥 이동
                    if (!visited[nx][ny][breaksLeft]) {
                        visited[nx][ny][breaksLeft] = true;
                        q.add(new int[]{nx, ny, dist + 1, breaksLeft});
                    }
                } else if (map[nx][ny] == 1 && breaksLeft > 0) {
                    // 벽: 기회가 남았으면 부수고 이동
                    if (!visited[nx][ny][breaksLeft - 1]) {
                        visited[nx][ny][breaksLeft - 1] = true;
                        q.add(new int[]{nx, ny, dist + 1, breaksLeft - 1});
                    }
                }
            }
        }

        // 목적지 도달 불가
        System.out.println(-1);
    }
}
```

## 시간복잡도 & 메모리 분석

- **상태 수**: N × M × (K+1) = 1,000 × 1,000 × 11 ≈ **11,000,000**
- **각 상태에서 탐색**: 최대 4방향
- **시간복잡도**: O(N × M × K) ≈ 1천만 — 충분히 빠릅니다
- **메모리**: `boolean[1000][1000][11]` ≈ 11MB — 제한 내 OK

## BFS vs DFS 선택 이유

이 문제는 BFS를 선택해야 합니다.

- BFS는 **최단 거리를 보장**합니다 (같은 비용의 이동이므로)
- DFS는 경로를 끝까지 탐색 후 비교하므로, 최단 거리를 구하려면 모든 경로를 탐색해야 합니다 → 시간 초과

## 배운 점

BFS에서 "상태"를 어떻게 정의하느냐가 매우 중요합니다. 단순히 (위치)가 아니라, 문제의 조건에 따라 상태에 추가 정보를 포함시켜야 합니다.

- 이 문제: `(위치, 남은 벽 부수기 횟수)`
- 유사 문제 패턴: `(위치, 아이템 보유 여부)`, `(위치, 특정 조건 달성 여부)` 등

상태가 복잡할수록 visited 배열의 차원도 늘어납니다. 상태 공간이 너무 크면 메모리 초과가 발생하므로, 문제의 제약 조건을 꼭 확인해야 합니다.
