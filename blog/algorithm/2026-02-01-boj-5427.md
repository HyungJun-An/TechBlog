불이 번지는 속도와 상근이의 이동 속도를 동기화하여 탈출 가능한 최단 시간을 구하는 BFS 시뮬레이션 풀이입니다. 이중 BFS를 1초 단위로 동기화하는 방법이 핵심입니다.

<!-- truncate -->

## 문제 설명

상근이가 건물 안에 갇혀 있고 불이 번지고 있습니다. 건물 밖으로 탈출하는 최단 시간을 구합니다.

**핵심 규칙**
- 불은 매 초 인접한 빈 칸으로 번집니다
- 상근이도 매 초 인접한 칸으로 이동할 수 있습니다
- 상근이는 불이 있는 칸, 혹은 불이 옮겨붙으려는 칸으로는 이동할 수 없습니다
- 지도 경계 밖으로 나가면 탈출 성공

[백준 5427번](https://www.acmicpc.net/problem/5427)

## 핵심 전략: 이중 BFS + 시간 동기화

불과 상근이의 이동을 **동일한 시간 단위로 동기화**해서 처리하는 것이 핵심입니다.

매 초마다:
1. **불 먼저 번짐** (현재 시간 기준의 불 위치들만 처리)
2. **상근이 이동** (현재 시간 기준의 위치들만 처리)

이를 구현하는 방법은 현재 큐의 사이즈만큼만 처리하는 것입니다.

## 풀이 코드

```java
import java.util.*;
import java.io.*;

public class Main {
    static int[] dx = {0, 1, 0, -1};
    static int[] dy = {1, 0, -1, 0};

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int T = Integer.parseInt(br.readLine().trim());
        StringBuilder sb = new StringBuilder();

        while (T-- > 0) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            int w = Integer.parseInt(st.nextToken());
            int h = Integer.parseInt(st.nextToken());

            char[][] map = new char[h][w];
            Queue<int[]> fireQ = new LinkedList<>();
            Queue<int[]> personQ = new LinkedList<>();
            boolean[][] visited = new boolean[h][w];

            for (int i = 0; i < h; i++) {
                String line = br.readLine();
                for (int j = 0; j < w; j++) {
                    map[i][j] = line.charAt(j);
                    if (map[i][j] == '*') {
                        fireQ.add(new int[]{i, j});
                    } else if (map[i][j] == '@') {
                        personQ.add(new int[]{i, j});
                        visited[i][j] = true;
                    }
                }
            }

            int time = 0;
            boolean escaped = false;

            outer:
            while (!personQ.isEmpty()) {
                time++;

                // 1. 불 번지기 (현재 큐 사이즈만큼만 처리)
                int fireSize = fireQ.size();
                for (int i = 0; i < fireSize; i++) {
                    int[] cur = fireQ.poll();
                    for (int d = 0; d < 4; d++) {
                        int nx = cur[0] + dx[d];
                        int ny = cur[1] + dy[d];
                        if (nx < 0 || nx >= h || ny < 0 || ny >= w) continue;
                        if (map[nx][ny] == '#' || map[nx][ny] == '*') continue;
                        map[nx][ny] = '*'; // 불로 표시
                        fireQ.add(new int[]{nx, ny});
                    }
                }

                // 2. 상근이 이동 (현재 큐 사이즈만큼만 처리)
                int personSize = personQ.size();
                for (int i = 0; i < personSize; i++) {
                    int[] cur = personQ.poll();
                    for (int d = 0; d < 4; d++) {
                        int nx = cur[0] + dx[d];
                        int ny = cur[1] + dy[d];

                        // 경계 밖 = 탈출!
                        if (nx < 0 || nx >= h || ny < 0 || ny >= w) {
                            escaped = true;
                            break outer;
                        }

                        if (visited[nx][ny]) continue;
                        if (map[nx][ny] == '#' || map[nx][ny] == '*') continue;

                        visited[nx][ny] = true;
                        personQ.add(new int[]{nx, ny});
                    }
                }
            }

            sb.append(escaped ? time : "IMPOSSIBLE").append('\n');
        }

        System.out.print(sb);
    }
}
```

## 핵심 구현 포인트

### 큐 사이즈만큼만 처리하는 이유

```java
int fireSize = fireQ.size();
for (int i = 0; i < fireSize; i++) {  // 현재 시간의 불만 처리
    // ...
}
```

만약 이 제한이 없으면 루프 안에서 새로 추가된 칸(다음 시간의 불)까지 같은 이터레이션에서 처리됩니다. 이렇게 하면 불이 한 번에 여러 칸씩 번지게 되어 잘못된 결과가 나옵니다.

### 불을 map에 직접 표시

불이 번진 칸을 map에 `'*'`으로 기록합니다. 덕분에 별도의 불 방문 배열을 만들 필요 없이, 상근이가 이동할 때 `map[nx][ny] == '*'` 조건 하나로 불을 피할 수 있습니다.

### 탈출 조건

경계 밖(`nx < 0 || nx >= h || ny < 0 || ny >= w`)으로 나가면 즉시 탈출 성공입니다. 인덱스 범위를 체크하는 일반적인 BFS와 달리, 여기서는 **경계 밖이 탈출을 의미**하므로 바깥으로 나가는 조건을 먼저 체크합니다.

## 시간복잡도

- V = w × h (최대 1,000,000)
- E = 4V
- **O(V + E) = O(V)** — 충분히 빠릅니다.

## 배운 점

BFS에서 여러 출발점을 동시에 처리하는 **멀티소스 BFS** 패턴을 배웠습니다. 불의 모든 위치를 시작부터 큐에 넣고 동시에 퍼트리는 방식으로, 각 칸마다 불이 도달하는 최솟값을 구할 수 있습니다. 이 패턴은 비슷한 유형의 문제(토마토 익히기 등)에서도 자주 사용됩니다.
