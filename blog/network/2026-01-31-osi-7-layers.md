## OSI 7계층이란?

ISO(국제표준화기구)에서 정의한 네트워크 표준 모델로, **통신 과정을 7개의 계층으로 나누어** 각 계층이 독립적으로 역할을 수행하도록 설계되었습니다.

```
7. 응용 계층 (Application)   ← 사용자와 가장 가까운 계층
6. 표현 계층 (Presentation)
5. 세션 계층 (Session)
4. 전송 계층 (Transport)
3. 네트워크 계층 (Network)
2. 데이터링크 계층 (Data Link)
1. 물리 계층 (Physical)      ← 실제 물리적 연결
```

## 각 계층 상세 설명

### 1계층 - 물리 계층 (Physical)
비트(0과 1) 단위의 데이터를 전기 신호로 변환하여 전송합니다.
- 장비: 케이블, 허브, 리피터
- 단위: **비트 (Bit)**

### 2계층 - 데이터링크 계층 (Data Link)
물리적 연결을 통한 신뢰성 있는 데이터 전송을 담당합니다. **MAC 주소**로 노드를 식별합니다.
- 장비: 스위치, 브릿지
- 단위: **프레임 (Frame)**
- 프로토콜: Ethernet, Wi-Fi (802.11)

### 3계층 - 네트워크 계층 (Network)
**IP 주소**를 기반으로 경로를 선택(라우팅)합니다.
- 장비: 라우터
- 단위: **패킷 (Packet)**
- 프로토콜: IP, ICMP, ARP

### 4계층 - 전송 계층 (Transport)
종단(End-to-End) 간 신뢰성 있는 데이터 전송을 보장합니다. **포트 번호**로 프로세스를 구분합니다.
- 단위: **세그먼트 (Segment)**
- 프로토콜: **TCP, UDP**

### 5계층 - 세션 계층 (Session)
통신 세션의 시작, 유지, 종료를 관리합니다.
- 프로토콜: TLS, NetBIOS

### 6계층 - 표현 계층 (Presentation)
데이터 형식 변환, 암호화, 압축을 담당합니다.
- 포맷: JPEG, MP4, SSL/TLS 암호화

### 7계층 - 응용 계층 (Application)
사용자와 직접 상호작용하는 서비스입니다.
- 프로토콜: HTTP, HTTPS, FTP, SMTP, DNS

## TCP vs UDP

### TCP (전송 제어 프로토콜)

연결을 먼저 수립한 후 통신하는 **연결 지향형** 프로토콜입니다.

**주요 특성**
- 연결이 성공해야만 통신 가능
- 각 바이트에 번호를 부여하여 순서 유지 및 재전송 지원
- 1:1 통신(유니캐스트)만 가능
- 속도는 느리지만 **신뢰성이 높음**

**연결 과정 (3-way Handshake)**
```
Client          Server
  │── SYN ────►│
  │◄── SYN+ACK─│
  │── ACK ────►│
  │   연결 완료  │
```

**해제 과정 (4-way Handshake)**
```
Client          Server
  │── FIN ────►│
  │◄── ACK ────│
  │◄── FIN ────│
  │── ACK ────►│
  │   연결 종료  │
```

**흐름 제어 & 혼잡 제어**: 수신자가 처리할 수 있는 속도와 네트워크 상태에 맞춰 전송 속도를 조절합니다.

### UDP (사용자 데이터그램 프로토콜)

연결 없이 통신하는 **비연결형** 프로토콜입니다.

**주요 특성**
- 연결 수립 없이 바로 전송
- 재전송 미지원, 순서 보장 없음
- 유니캐스트, 브로드캐스트, **멀티캐스트** 지원
- 오버헤드가 낮아 **전송 속도가 빠름**

**사용 사례**: DNS, 게임, 스트리밍, VoIP (실시간성이 중요하고 약간의 손실이 허용되는 경우)

| 항목 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결 지향 | 비연결 |
| 신뢰성 | 높음 | 낮음 |
| 순서 보장 | ✅ | ❌ |
| 속도 | 느림 | 빠름 |
| 사용 | HTTP, FTP, DB | DNS, 게임, 스트리밍 |

## IP (인터넷 프로토콜)

IP 주소는 네트워크에서 **호스트를 식별**하고, 패킷이 올바른 목적지로 도달하도록 경로를 안내합니다.

### 비신뢰성 (Unreliable)
패킷 전달을 보장하지 않습니다. 패킷이 유실되거나 순서가 뒤바뀔 수 있습니다.
→ ICMP를 통해 오류를 감지할 수 있습니다 (`ping` 명령어가 ICMP를 사용).

### 비연결성 (Connectionless)
대상 호스트의 상태를 확인하지 않고 그냥 전송합니다. 연결 수립 과정이 없으므로 빠르지만, 신뢰성은 상위 계층(TCP)에 맡깁니다.

## DNS (도메인 네임 시스템)

**도메인 주소 → IP 주소** 변환을 담당하는 시스템입니다.

```
브라우저에서 www.example.com 입력
        ↓
로컬 캐시 확인
        ↓ (캐시 없을 경우)
ISP DNS 리졸버에 쿼리
        ↓
루트 DNS → TLD DNS (.com) → 권한 있는 DNS
        ↓
IP 주소 반환 → 브라우저가 해당 IP로 접속
```

DNS는 **UDP 포트 53**을 사용하며, 응답 크기가 클 경우 TCP로 전환됩니다.

## 로드밸런서

**서버의 부하를 여러 서버로 분산**시켜 가용성과 성능을 높이는 시스템입니다.

### L4 vs L7

| 항목 | L4 로드밸런서 | L7 로드밸런서 |
|------|-------------|-------------|
| 계층 | 전송 계층 (4계층) | 응용 계층 (7계층) |
| 기준 | IP + 포트 | URL, HTTP 헤더, 쿠키 |
| 속도 | 빠름 | 상대적으로 느림 |
| 세밀함 | 낮음 | 높음 (콘텐츠 기반 라우팅) |
| 비용 | 낮음 | 높음 |

### 계층별 분류
- **L2**: MAC 주소 기반 (스위치)
- **L3**: IP 주소 기반 (라우터)
- **L4**: TCP/UDP 포트 기반
- **L7**: HTTP 경로, 헤더, 페이로드 기반

**핵심**: 상위 계층으로 갈수록 더 세밀한 분산이 가능하지만, 처리 비용이 증가합니다.

실무에서는 정적 파일(이미지, JS)은 CDN이나 L4로, API 요청은 L7 로드밸런서로 라우팅하는 조합을 많이 사용합니다.
