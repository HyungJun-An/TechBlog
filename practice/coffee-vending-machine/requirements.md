# 커피 자판기 — 요구사항 분석서

---

## 액터 (Actor)

| 액터 | 설명 |
|------|------|
| **일반 사용자** | 음료를 선택하고 결제하는 주체 |
| **관리자** | 비밀번호로 인증 후 재고 보충·매출 조회 |

---

## 기능 요구사항

### [사용자]

| ID | 요구사항 |
|----|---------|
| U-01 | 판매 가능한 음료 목록과 가격을 확인할 수 있다 |
| U-02 | 음료를 선택할 수 있다 (재고 부족 시 선택 불가 표시) |
| U-03 | 결제 수단을 선택할 수 있다 (현금 / 카드) |
| U-04 | 현금 결제 시 금액을 투입하고, 잔액이 부족하면 추가 투입할 수 있다 |
| U-05 | 현금 결제 완료 후 거스름돈을 반환받는다 |
| U-06 | 카드 결제 시 카드 번호를 입력해 승인한다 |
| U-07 | 결제 성공 시 음료가 제조되고 출력된다 |
| U-08 | 결제 취소 시 투입한 현금이 반환된다 |

### [관리자]

| ID | 요구사항 |
|----|---------|
| A-01 | 비밀번호를 입력해 관리자 모드로 진입할 수 있다 |
| A-02 | 음료별 재고를 보충할 수 있다 |
| A-03 | 현재 음료별 재고 현황을 조회할 수 있다 |
| A-04 | 총 매출을 조회할 수 있다 |

## 비기능 요구사항

| ID | 요구사항 |
|----|---------|
| N-01 | 잘못된 입력(숫자 외 문자 등)에 대해 예외 처리를 한다 |
| N-02 | 관리자 비밀번호 불일치 시 접근을 거부한다 |
| N-03 | 재고 소진 시 즉시 판매 불가로 처리한다 |

---

## 설계 원칙 및 적용 패턴

| 패턴 / 원칙 | 적용 위치 |
|------------|---------|
| 추상 클래스 | `Beverage` — 공통 필드(name, price)와 공통 메서드 공유 |
| 인터페이스 | `PaymentStrategy` — 결제 수단 계약 정의 |
| 전략 패턴 | `VendingMachine`이 `PaymentStrategy`를 주입받아 런타임에 교체 |
| 다형성 | `Beverage[]`로 모든 음료를 일괄 처리 |
